NISA口座運用 代理制度判定エンジン

証券会社からの回答を、再来週の購入実行ルートへ変換するための判定表です。

判定の目的

確認NISA口座で代理人注文が可能か。
期限再来週の購入予定に間に合うか。
代替間に合わない場合の本人発注型を用意する。
継続中期でIFA・ラップ・代理制度へ移行する。

入力する回答項目

入力項目 選択肢 重み 理由
代理人制度 あり / なし / 不明 最重要 正式制度がない場合は、代理操作案として採用できない。
NISA対象可否 NISA注文可 / 照会のみ / NISA不可 / 不明 最重要 NISA口座で注文できなければ、今回のNISA購入解決策にならない。
権限範囲 注文・売却・決済まで / 注文のみ / 照会のみ / 不明 高 購入後の売却・リスク対応まで見据えると、注文だけでは弱い。
開始時期 再来週前 / 1か月以内 / 1か月超 / 不明 高 再来週購入に間に合うかを決める。
書類負担 軽い / 普通 / 重い / 公正証書等が必要 中 顧客の実務負担と導入速度に影響する。

ルート判定

ルート 条件 再来週の対応 顧客満足 注意点
A 正式代理 代理人制度あり。NISA注文可。注文・決済権限あり。再来週前に開始できる。 正式代理フローで初回購入を準備。 高。本人負担を下げやすい。 証券会社所定手続と本人同意を記録する。
B 本人発注 代理制度なし、または再来週に間に合わない。ただし本人が発注できる。 注文票を作成し、本人スマホ・本人ログイン・本人発注で進める。 中。手間は残るが購入予定は維持できる。 発注画面を第三者が操作しない。
C 中期移行 代理制度はあるが開始に時間がかかる。登録業者連携やラップの方が現実的。 初回は本人発注型。翌月以降に正式制度へ移行。 中〜高。継続運用の体制を作れる。 初回購入と中期制度整備を分けて説明する。
D 購入延期 代理制度なし。本人発注も難しい。口座区分や本人確認も未整備。 その本人分は購入しない。 短期満足は低いが、誤った運用を避ける。 口座体制が整うまで待つ。

証券会社回答を入れた後の出力例

確認先 仮回答 判定 実務対応
三菱UFJ eスマート証券 代理人申請あり。NISA対象可否は要確認。 優先確認 最初に電話確認。NISA注文可ならA、不可ならB/C。
SBI証券 未確認。 確認待ち ネット証券の低コスト候補として確認。不可なら本人発注型。
対面証券 家族サポート証券口座等の確認対象。 中期確認 再来週に間に合わない場合でも継続運用案として残す。
NISAラップ 個別株ではなく投信/ラップで運用。 安全代替 本人操作負担を下げたい顧客向けに別メニュー化。
