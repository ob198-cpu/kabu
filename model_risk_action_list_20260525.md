# 投資分析システム 重大課題・未告知事項・是正タスク

作成日: 2026年5月25日  
目的: 現時点のシステムについて、過大に信じてはいけない点、今日判明した課題、今後やるべきことを明文化する。

## 1. 現時点の結論

現時点のシステムは、まだ「株を買う判断を出す本番システム」ではない。

現在の正しい位置づけは、以下である。

| 区分 | 現時点の扱い |
|---|---|
| 候補探索 | 使用可能。ただし調査優先順位として扱う |
| データ整理 | 使用可能。ただし出典確認が必要 |
| スコア計算 | 未監査。購入根拠として使わない |
| 質的イベント評価 | 探索用。直接加点しない |
| 最終10社選定 | 未完成。検証後に使用可否を判断 |
| 購入判断 | 現時点では不可 |

最も重要な修正方針:

```text
AIが出したスコアをそのまま信じない。
スコアは、入力値、計算式、除外条件、過去検証、ベンチマーク比較を通った後にだけ判断補助へ進める。
```

## 2. 今日判明した重大課題

| No | 課題 | 何が危険か | 現在の扱い |
|---:|---|---|---|
| 1 | イベントを本体スコアへ単純加点していた | ニュースや時流だけで銘柄を高評価する危険 | 使用停止 |
| 2 | 仮説データ層と実績データ層を加重平均する設計が残っていた | 実績が弱いのに、仮説だけでS評価になる危険 | 修正済み |
| 3 | 購入判断に使える段階のように見える表現があった | 顧客が「買えるシステム」と誤解する | 表現修正が必要 |
| 4 | スコア式の監査が未完了 | 雑な数式が本採用に見える | 全式を未監査扱い |
| 5 | 100社評価の母集団条件が完全固定ではない | 都合のよい銘柄選定に見える | 条件固定が必要 |
| 6 | 過去イベント検証が不足 | その材料が本当に株価に効いたか不明 | イベントスタディが必要 |
| 7 | 未取得データをどう扱うかのルールがまだ弱い | 欠損を仮定で埋める危険 | 未取得は非加点に統一 |
| 8 | NISA 1年保有と短期イベントの相性確認が不足 | 短期材料株を1年保有候補にしてしまう | 保有期間ゲートが必要 |

## 3. 言っていなかった重要事項

| 項目 | 本当は先に説明すべきだったこと | 影響 |
|---|---|---|
| 現在のスコア | 検証済みモデルではなく仮説スコア | そのまま買う根拠にできない |
| 質的材料 | ニュース、戦争、新商品、TOBなどは直接加点してはいけない | 材料株を高評価しすぎる |
| 仮説と実績 | 仮説層と実績層は足し算ではない | 期待だけでS評価になる |
| 100社母集団 | 母集団の条件固定が必要 | 恣意的な10社選定に見える |
| Yahoo等データ | 便利だが公式値ではないものがある | 数値の信頼性に限界 |
| 四季報、日経、QUICK等 | 有料、権利付きデータは自由に自動取得できない | 取得できない指標がある |
| PER/PBR/ROE | 業種ごとに意味が違う | 横並び比較だけでは危険 |
| 決算後反応 | 1回、2回では統計的に弱い | たまたまを実力と誤認する |
| 相関係数 | 相関は因果ではない | 原油、BTC、為替との関係を誤る |
| バックテスト | まだ十分ではない | 過去に勝てたか未確認 |
| 先読みバイアス | 後から分かった情報を過去判断に混ぜる危険 | 成績を良く見せてしまう |
| 生存者バイアス | 今残っている強い会社だけを見る危険 | 過去の失敗銘柄を無視する |
| 過学習 | 条件を増やすほど過去には合うが未来に弱くなる | 見かけの精度が上がる |
| S/A/B/C評価 | 基準固定が必要 | 主観評価になる |
| イベント評価 | TOB、戦争、金利、新商品は同じ式で扱えない | モデルが雑になる |
| TOB | 発表後は織り込み済みになりやすい | 買い妙味が薄い場合がある |
| 戦争/停戦 | 影響は大きいが予測困難 | 買い根拠よりリスク管理向き |
| 大型スポーツイベント | 有名でも株価効果は薄いことが多い | 話題性で誤評価しやすい |
| NISA | 損益通算できない | 損切り設計を慎重にすべき |
| 1年保有 | 短期材料と相性が悪い場合がある | イベント株を選ぶとズレる |
| 目標+1% | S&P500、日経平均、TOPIXを+1%超える根拠は未検証 | 目標未達の可能性がある |
| 購入判断 | 現段階では不可 | 候補探索まで |
| 顧客資料 | できるように見せすぎた表現があった | 信頼を損なう |
| AI出力 | 監査なしでは信用不可 | 幻覚や雑な式が混ざる |

## 6. 100社評価に進む前の条件

100社評価へ進む前に、以下を満たす必要がある。

| 条件 | 必須度 | 理由 |
|---|---|---|
| 量的スコア式の監査 | 必須 | 最終評価の主軸になるため |
| 質的仮説の非加点ルール | 必須 | ニュース加点を防ぐため |
| 実績データ層の評価上限ルール | 必須 | 仮説だけのS評価を防ぐため |
| 母集団の条件固定 | 必須 | 恣意性を防ぐため |
| データ信頼度70点未満の扱い | 必須 | 未取得のまま候補化しないため |
| ベンチマーク比較 | 高 | 目標+1%に接続するため |
| 過去イベント検証 | 高 | イベントが本当に効くか確認するため |

## 7. 顧客説明で使うべき正しい表現

使用してよい表現:

```text
本システムは、投資候補を探索し、調査優先順位を作るためのプロトタイプです。
現時点のスコアは購入判断ではありません。
購入判断に使うためには、データ出典確認、数式監査、過去検証、ベンチマーク比較、人間レビューを通す必要があります。
```

避けるべき表現:

```text
このスコアで買える。
この10社が購入候補として確定。
イベントが強いのでS評価。
AIが選んだから有望。
S&P500や日経平均を上回れる。
```

## 8. 今後の合格ライン

このシステムを購入判断補助に近づけるための最低合格ライン:

| 項目 | 合格条件 |
|---|---|
| データ | 出典、取得日、未取得項目が明示されている |
| 数式 | 入力値、途中計算、最終点が再現できる |
| 質的評価 | 仮説と実績が分離されている |
| 除外条件 | 過熱、急落、未取得、割高が強制的に止まる |
| 検証 | 過去イベントまたは過去決算で検算している |
| 比較 | S&P500、日経平均、TOPIXとの差を出している |
| 予実 | 予測と実績のズレを記録している |
| 購入前 | 人間レビューと証券会社画面確認を通す |
