# +1%超過目標 実運用向け最小検証モデル

作成日: 2026年5月25日  
目的: 論文発表ではなく、証券会社の無難運用、S&P500、日経平均、TOPIXを最低+1%上回ることを目標にした、実運用向けの最小検証モデルを定義する。

## 1. ゴール

本プロジェクトのゴールは、学術論文ではない。

ゴールは以下である。

```text
NISA 1年保有を前提に、証券会社の無難運用、S&P500、日経平均、TOPIXを
最低+1%上回る可能性がある候補を、実データと検証ルールで10社程度まで絞る。
```

重要:

```text
論文水準の厳密さは、恥ずかしい論理ミスを避けるために使う。
目的は、実運用で+1%超過を狙える候補選定である。
```

## 2. 比較対象

| 比較対象 | 目的 | 判定 |
|---|---|---|
| 証券会社の無難運用 | 顧客が任せた場合との比較 | これを下回るなら個別株選定の意味が薄い |
| S&P500投信 | 世界株・米国株の標準代替 | 円建てで比較 |
| 日経平均連動投信 | 日本大型株の代表 | 日本株個別選定の最低比較対象 |
| TOPIX連動投信 | 日本株全体の代表 | 偏りの少ない比較対象 |

合格ライン:

```text
候補10社ポートフォリオの1年リターン - 比較対象の1年リターン >= +1%
```

## 4. 最小構成

本番に近づけるための最小構成は以下。

| 構成 | 内容 | 必要性 |
|---|---|---|
| 100社母集団 | 業績、テーマ、流動性で候補を広く作る | 恣意性を防ぐ |
| 量的スコア | 売上、利益、ROE、PER/PBR、下落率、決算後反応 | 主軸 |
| 質的イベント層 | 新商品、金利、戦争、TOB等を候補発見に使う | 補助 |
| 実績イベント層 | 過去イベント後に本当に動いたか確認 | 質的評価の上限 |
| 除外ゲート | 未取得、過熱、急落、割高、流動性不足を止める | 必須 |
| ベンチマーク比較 | S&P500、日経平均、TOPIXとの差を見る | 必須 |
| 予実記録 | 予測と実績のズレを保存 | 改善に必要 |

## 5. 基本フロー

```text
1. 100社前後の母集団を作る
2. 量的データで一次評価する
3. 未取得、過熱、急落、割高、流動性不足を除外する
4. 質的イベントで追加調査の優先順位をつける
5. 実績データがあるイベントだけ評価に残す
6. 上位20社を作る
7. 10社まで絞る
8. S&P500、日経平均、TOPIX、証券会社運用と比較する
9. +1%超過が狙えないなら個別株比率を下げる
10. 実績を記録し、外れた理由を次回修正に使う
```

## 6. スコアの考え方

### 6.1 主軸は量的データ

```text
購入候補スコア = 量的スコア中心
```

量的スコアに入れるもの:

| 項目 | 例 |
|---|---|
| 成長 | 売上成長率、利益成長率、EPS成長率 |
| 収益性 | ROE、営業利益率、FCF |
| 割高/割安 | PER、PBR、業種中央値との比較 |
| 下落耐性 | 60日最大下落率、ボラティリティ |
| 株価反応 | 決算後1日、5日、20日の対指数超過リターン |
| 流動性 | 出来高、売買代金 |
| 信頼度 | 公式データ確認率、未取得数 |

### 6.2 質的データは直接加点しない

```text
質的仮説 = 候補発見と調査優先度
質的実績 = その仮説を採用できるかの上限判定
```

例:

| 状態 | 扱い |
|---|---|
| 仮説S × 実績S | 量的ゲート通過後に強い候補 |
| 仮説S × 実績C | 話題は強いが候補化しない |
| 仮説C × 実績S | 再調査。別要因の可能性 |
| 実績なし | 購入候補スコアに入れない |

## 7. 除外ゲート

以下に該当する銘柄は、点数が高くても初回候補にしない。

| ゲート | 除外/保留条件 |
|---|---|
| 未取得 | PER/PBR/ROE、売上成長、利益成長、決算後反応の重要項目が不足 |
| 過熱 | 1年上昇率が極端、かつ利益成長で説明できない |
| 急落 | 60日最大下落率が大きい |
| 割高 | PER/PBRが業種中央値から大きく外れる |
| 反応悪化 | 決算後の対指数リターンが弱い |
| 流動性不足 | 売買代金が小さく、実運用で再現しにくい |
| NISA不適合 | 短期材料依存で1年保有に向かない |

## 8. 10社候補の条件

10社候補に入れる最低条件:

| 条件 | 最低ライン |
|---|---|
| 量的評価 | B以上 |
| データ信頼度 | 70点以上 |
| ハードゲート | 該当なし |
| 流動性 | 実運用可能 |
| ベンチマーク期待 | 比較対象を+1%以上上回る可能性が説明できる |
| 質的評価 | B以上が望ましい。ただし直接加点しない |

## 9. 買わない判断

以下の場合は、10社候補があっても買わない。

| 条件 | 行動 |
|---|---|
| S&P500、日経、TOPIXに勝てる根拠がない | 個別株比率を下げる |
| 市場全体が急落局面 | 購入延期 |
| 米金利急騰、円高ショック、VIX急騰 | 購入延期 |
| 10社の多くが同一テーマに偏る | 分散不足として見直し |
| データ信頼度が低い | 調査継続 |
| 購入理由がイベント期待だけ | 購入不可 |

## 10. 実績記録

購入またはテスト候補にした場合、必ず以下を記録する。

| 記録項目 | 内容 |
|---|---|
| 選定日 | いつ選んだか |
| 選定時価格 | 当時の株価 |
| 量的スコア | 採用時点の点数 |
| 質的仮説 | どのイベントに関係するか |
| 実績根拠 | 過去反応、KPI、相関など |
| 除外ゲート | 通過/非通過 |
| 比較対象 | S&P500、日経、TOPIX |
| 1か月後 | 実績リターン |
| 3か月後 | 実績リターン |
| 6か月後 | 実績リターン |
| 12か月後 | 実績リターン |
| 予測誤差 | 予測との差 |
| 外れた理由 | 金利、決算、テーマ失速、過熱など |

## 12. 顧客向けの正しい説明

使うべき説明:

```text

現時点では購入判断ではなく、100社前後から調査候補を絞り、量的データとイベント実績で10社程度に絞る段階です。
購入可否は、6月の市場イベント、最新決算、ベンチマーク比較、除外ゲートを通過した後に判断します。
```

```text

この10社で確定。

```
