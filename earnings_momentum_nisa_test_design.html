業績モメンタム母集団・時流選定・NISAテスト設計

目的: 数字で候補群を作り、時流テーマで絞り、NISA 1年保有テストで判断補助として検証する

作成日時: 2026/05/22 13:09

1. 結論

100社という数を先に決めるのではなく、直近2回決算で業績が強い会社を抽出し、結果として80〜120社程度の母集団を作る方針に変更します。

全上場企業まず広く見る。最初から主観で10社を選ばない。
最低条件で除外流動性不足、データ不足、赤字拡大、一過性要因を確認。
業績上位群直近2回決算の売上・利益・EPS・予想修正で順位化。
時流で絞るAI、電力、防衛、金利、円安など業績に効くテーマを確認。
NISAテスト候補8〜10社を1年保有前提で記録し、指数と比較。
この資料は購入銘柄を確定するものではありません。購入前に、公式資料で数値を検算し、比較指数とルールを固定してから判断します。

2. 母集団の作り方

phaserulenumeric_rulereason
1. 対象企業の土台全上場企業から始める。ただし、最初から全銘柄を同列に計算せず、データ取得可否と流動性で足切りする。売買代金・出来高が極端に少ない銘柄、直近2回決算が取得できない銘柄は一次除外。買えない銘柄、数字を検算できない銘柄を上位に入れないため。
2. 業績モメンタム抽出直近2回の決算で、売上・営業利益・EPS・会社予想修正が強い銘柄を順位化する。各指標を0〜100点のパーセンタイルへ変換し、合計点で上位約10%を抽出。単発の好決算ではなく、2回連続で数字が改善している会社を残すため。
3. 約100社への調整上位10%が多すぎる場合は流動性・財務安全性・異常値で絞り、少なすぎる場合は上位15%まで広げる。目安は80〜120社。100社ぴったりを目的にしない。意味のある母集団を優先し、数字合わせを避けるため。
4. 時流テーマ分類AI、電力、防衛、金利、円安、医療、インバウンドなどのテーマに分類する。テーマはニュース名ではなく、売上・受注・利益・業界統計に数字が出ているものを優先。話題性だけでなく、業績に効いている銘柄を選ぶため。
5. 公式資料で検算候補銘柄は決算短信、IR、有報、EDINET、J-Quants等で数値を確認してからテスト候補にする。根拠数値と取得元を記録できない銘柄は、購入検討候補ではなく観察候補に留める。AIが数字を推測した状態で銘柄を確定しないため。

3. 業績モメンタムスコア

各指標はそのまま足しません。業種差や異常値を抑えるため、同じ母集団内で0〜100点の順位点へ変換してから合計します。

業績モメンタムスコア = 売上成長20点 + 営業利益成長25点 + EPS成長15点 + 会社予想修正15点 + 利益率改善10点 + 決算後株価反応10点 + 流動性5点
itemweightcalculationcaution
売上成長20点直近決算YoYと1つ前の決算YoYをパーセンタイル化前年の落ち込みからの反動だけなら異常値フラグを付ける。
営業利益成長25点直近決算YoYと1つ前の決算YoYをパーセンタイル化赤字から黒字化は率が暴れやすいため、金額改善と併用する。
EPS成長15点EPSまたは純利益のYoYをパーセンタイル化一過性利益、税効果、特別利益は注記で確認する。
会社予想修正15点(新予想 - 旧予想) ÷ 旧予想上方修正は強い材料。下方修正は原則減点。
利益率改善10点営業利益率の前年同期差、または直近推移売上増でも利益率が悪化している銘柄を避ける。
決算後株価反応10点決算発表後1日・5日・20日の騰落率既に買われすぎた銘柄は買い時スコアで調整する。
流動性5点平均出来高、売買代金、スプレッドの確認NISAで保有する前提でも、売れない銘柄は避ける。

4. 時流で絞る基準

需要

AI、電力、防衛、医療、インバウンドなどで需要が増えているか。

数字

売上、受注、利益率、会社計画、業界統計に反映されているか。

株価

すでに買われすぎていないか。決算後の反応と下落余地を見る。

時流テーマは「話題だから採用」ではなく、「テーマが業績に出ている、または出る根拠がある」ものだけを採用します。

5. NISAテストの目的と手続き

NISAテストの目的は、1年で勝てることを証明することではなく、銘柄選定システムが実際の判断補助として使えるかを検証することです。

stepprocedurepurpose
1. ルール固定銘柄抽出前に、使う指標・重み・除外条件・比較指数を固定する。後から都合よく銘柄や数式を変えたように見えないようにする。
2. 母集団作成直近2回決算の業績モメンタムで80〜120社の候補群を作る。最初から主観で10社を選ばず、数字で候補群を作る。
3. 時流テーマ確認テーマ別に、業界統計・会社資料・受注・売上寄与の数字を確認する。ニュースではなく、業績に効くテーマか確認する。
4. 公式資料検算上位候補を決算短信・IR・EDINET/J-Quantsで検算し、根拠表へ記録する。取得元不明の数値を使った判断を避ける。
5. テスト候補決定8〜10社に絞り、各銘柄の採用理由、見送り条件、下落時対応を記録する。購入判断前に、なぜ選んだか、いつ見直すかを明確にする。
6. NISAテスト1年保有を前提に、月次で指数比較、最大下落、決算変化、ルール遵守を記録する。投資判断補助システムとして実用になるか検証する。

判定の考え方

  • 主目標: S&P500円建て相当、TOPIX、日経平均など主要比較対象を1年後に+1%以上上回ることを目標にする。
  • 3か月時点: 勝敗判定ではなく、記録・ルール運用・下落時対応ができているかを確認する。
  • 途中見直し: 下方修正、重大リスク、指数急落、決算悪化など、事前に決めた条件でのみ行う。

8. 次工程で作る表

  • 直近2回決算入力表: 銘柄、決算期、売上、営業利益、EPS、会社予想、取得元URL。
  • 業績モメンタム計算表: 各指標の順位点、異常値フラグ、総合点。
  • 時流テーマ対応表: テーマ、根拠資料、業績寄与、候補銘柄。
  • NISAテスト記録表: 採用理由、購入判断日、比較指数、月次評価、予想との差。

取得元を記録できない数値は、スコアに混ぜず「補完待ち」として扱います。
