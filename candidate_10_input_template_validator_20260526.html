候補10社 CSV入力・検証テンプレート

候補10社を根拠付きで選ぶために、PER/PBR/ROE、同業比較、決算後反応、イベント実績を入力して、採点へ接続できる状態かを確認する画面です。

この画面で行うこと

入力対象33件10社分の不足確認項目
CSV操作対応読み込み、検証、出力
初期状態未接続未入力値は採点に使わない
購入判断0社6月イベント後に別判定

このテンプレートは、根拠データをそろえるための作業画面です。出典・基準日・数値がそろうまで、点数や購入判断には接続しません。

CSV操作

まだCSVは読み込まれていません。まずテンプレートを保存し、値を入力してから読み込んでください。

入力種別の内訳

event: 8件

metric_1=イベント後株価反応 / metric_2=実績評価S-A-B-C / metric_3=関連度 / metric_4=継続期間

接続条件: period_or_event_date、source_url、metric_1、metric_2が必須。仮説だけなら採点へ接続しない。

peer: 5件

metric_1=同業PER中央値 / metric_2=同業PBR中央値 / metric_3=同業ROE中央値 / metric_4=同業社数

接続条件: source_url、source_date、metric_1、metric_4が必須。同業社数が少ない場合は説明補助に止める。

valuation: 10件

metric_1=対象指標値 / metric_2=同業中央値 / metric_3=平均との差 / metric_4=採用可否

接続条件: source_url、source_date、metric_1、benchmark_or_basisが必須。業種比較の基準も記録する。

reaction: 7件

metric_1=1日超過リターン / metric_2=5日超過リターン / metric_3=20日超過リターン / metric_4=反応判定

接続条件: period_or_event_date、source_url、metric_1、metric_2、metric_3が必須。20日未到達なら未接続。

financial: 3件

metric_1=予想PERまたは実績PER / metric_2=EPS基準 / metric_3=PBR / metric_4=ROE

接続条件: source_url、source_date、metric_1が必須。出典不明なら採点へ接続しない。
