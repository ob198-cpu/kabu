# 投資分析モデル 学術的妥当性プロトコル

作成日: 2026年5月25日  
目的: 投資候補探索システムを「雰囲気のあるスコア」から、再現性・検証可能性・反証可能性を持つ研究設計へ引き上げる。

## 1. 基本方針

このシステムは、以下の順番で研究水準へ引き上げる。

```text
仮説を作る
→ 事前に評価式と除外条件を固定する
→ point-in-timeデータで検証する
→ 過去データでベンチマークに勝ったか確認する
→ 未知期間で外部検証する
→ 予測と実績の誤差を記録する
→ 採用/不採用を決める
```

重要:

```text
検証前の式は、購入判断ではなく研究仮説である。
```

## 2. 研究としての問い

本システムが検証すべき問いは、以下である。

| 問い | 検証方法 |
|---|---|
| 質的イベントは株価に影響するか | イベントスタディで異常リターンを測る |
| その影響は一時的か、1年保有に使えるか | 1日、5日、20日、60日、250営業日で比較 |
| 量的スコアは将来リターンを説明するか | ランキング別ポートフォリオで検証 |
| 質的データは量的データに上乗せ価値を持つか | 量的モデル単独 vs 量的+質的モデルで比較 |
| S&P500、日経平均、TOPIXを+1%以上上回れるか | 円建て・配当込み・コスト込みで比較 |

## 3. 使うべき学術的方法

| 方法 | 用途 | 参考 |
|---|---|---|
| イベントスタディ | TOB、上方修正、新商品、戦争、金利イベント後の異常リターン測定 | MacKinlay 1997, Brown & Warner 1985 |
| マーケットモデル | 個別株の通常リターンを市場リターンで説明し、異常リターンを出す | Event Study標準手法 |
| Fama-French/Carhart型ファクター補正 | サイズ、バリュー、モメンタムなど既知ファクターを控除 | Fama & French 1993, Carhart 1997 |
| モメンタム検証 | 過去リターンの継続性を検証 | Jegadeesh & Titman 1993 |
| 多重検定補正 | たくさん試した結果、偶然当たる問題を抑える | Harvey, Liu & Zhu 2016 |
| ウォークフォワード検証 | 過去で作った式を未来期間で検証 | 実運用に近い検証 |
| 予測誤差記録 | 外れ方を記録し、モデル改善に使う | 運用後評価 |

## 4. データ要件

| データ | 必須条件 | 理由 |
|---|---|---|
| 株価 | 調整後株価、出来高、売買代金、上場廃止銘柄を含むことが望ましい | 生存者バイアスを避ける |
| 財務 | 決算発表日ベースで使えるpoint-in-timeデータ | 先読みバイアスを避ける |
| PER/PBR/ROE | 取得日と計算根拠を保存 | 後から値が変わるため |
| イベント | 発生日、発表時刻、情報源、イベント分類 | イベント前後分析に必要 |
| ベンチマーク | TOPIX、日経平均、S&P500円建て、必要なら配当込み | 目標+1%判定に必要 |
| ファクター | 市場、サイズ、バリュー、モメンタム等 | 既知要因を控除するため |
| 取引コスト | スプレッド、手数料、為替、税制差 | 実運用成績に近づける |

## 5. バイアス対策

| バイアス | 内容 | 対策 |
|---|---|---|
| 先読みバイアス | 当時まだ知られていない情報を使う | 発表日、取得日を記録し、その日以降だけ使用 |
| 生存者バイアス | 今残っている会社だけで検証する | 可能なら上場廃止銘柄を含む |
| データスヌーピング | たくさん試して偶然当たった式を採用する | 事前登録、ホールドアウト、多重検定補正 |
| 過学習 | 過去に合いすぎて未来で弱い | 変数数を制限し、ウォークフォワード検証 |
| イベント重複 | 同時に複数イベントが起きる | 重複イベントを除外または統制変数化 |
| 流動性バイアス | 小型・薄商いで再現できない | 売買代金ゲート、スプレッド控除 |
| 公表時刻問題 | 引け後発表を当日リターンに混ぜる | 発表時刻でイベント日をずらす |

## 6. イベントスタディ設計

### 6.1 異常リターン

```text
AR_i,t = R_i,t - E(R_i,t)
```

通常リターンの候補:

```text
E(R_i,t) = α_i + β_i R_market,t
```

または、

```text
E(R_i,t) = α_i + β_m MKT_t + β_s SMB_t + β_h HML_t + β_mom MOM_t
```

### 6.2 累積異常リターン

```text
CAR_i,[a,b] = Σ AR_i,t
```

見る期間:

| 期間 | 意味 |
|---|---|
| [0,1] | 即時反応 |
| [0,5] | 短期反応 |
| [0,20] | 約1か月の反応 |
| [0,60] | 四半期程度の反応 |
| [0,250] | NISA 1年保有に近い反応 |

## 7. 質的データの扱い

質的データは直接加点しない。

| 層 | 役割 | スコアへの扱い |
|---|---|---|
| 仮説データ層 | 企業を探す入口 | 原則、直接加点しない |
| 実績データ層 | 過去に効いたか検証 | 質的評価の上限を決める |
| 量的データ層 | 最終判断の主軸 | 購入候補スコアの中心 |

ルール:

```text
仮説S × 実績C = 質的Sにしない
仮説C × 実績S = まず再調査
実績なし = 購入候補スコアへ反映しない
```

## 8. モデル比較

最低限、以下を比較する。

| モデル | 内容 | 目的 |
|---|---|---|
| Benchmark | TOPIX、日経平均、S&P500円建て | 何もしない場合との比較 |
| Quant Only | 量的データだけ | 基礎モデル |
| Event Evidence Only | 実績イベントだけ | イベントの有効性確認 |
| Quant + Event Evidence | 量的 + 実績イベント | 質的データが上乗せ価値を持つか確認 |
| Quant + Hypothesis | 量的 + 仮説 | 原則採用しない。危険性確認用 |

採用候補にできるのは、`Quant + Event Evidence` が `Quant Only` を未知期間で上回った場合だけ。

## 9. 採用基準

| 項目 | 採用基準 |
|---|---|
| 期待リターン | ベンチマークを年率または1年で+1%以上上回る |
| 勝率 | ランダム選定や同業平均より有意に高い |
| 最大下落 | ベンチマークより極端に悪くない |
| シャープ比 | ベンチマーク以上 |
| 多重検定 | 補正後も有意性が残る |
| 取引可能性 | 売買代金、スプレッド、単元、NISA制約を満たす |
| 再現性 | 学習期間だけでなく検証期間でも機能する |

## 10. 実装順序

| フェーズ | 内容 | 成果物 |
|---|---|---|
| 1 | 現行スコア式を全監査 | 採用/廃止/検証待ちリスト |
| 2 | 100社母集団の条件固定 | 母集団定義表 |
| 3 | point-in-timeデータ保存 | 入力データ台帳 |
| 4 | イベント分類表を固定 | イベントタクソノミー |
| 5 | イベントスタディ実装 | AR/CAR計算表 |
| 6 | 量的モデル実装 | Quant Onlyモデル |
| 7 | 質的実績モデル結合 | Quant + Event Evidenceモデル |
| 8 | ウォークフォワード検証 | 検証レポート |
| 9 | ベンチマーク比較 | +1%判定表 |
| 10 | 10社候補を出す | テスト候補リスト |

## 11. 論文水準に必要な追加条件

| 条件 | 必要性 |
|---|---|

| 事前仮説の固定 | 必須 |
| ホールドアウト期間 | 必須 |
| 多重検定補正 | 必須 |
