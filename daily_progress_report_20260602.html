2. 本日の実施内容

項目実施内容状態補足
実用パート整理 ホームと実用ダッシュボードを、目的から辿れる構成に整理しました。毎日見る画面、候補10社、6月イベント後の再判定、NISA枠、税制確認に入口を分けています。 反映済み 資料閲覧用と実務確認用を分ける方向で整理を継続します。
NISA枠配分候補 現在の10社候補について、NISA枠で使う価値があるかを別軸で確認する画面とCSVを作成しました。 反映済み 既存の銘柄スコアを上書きせず、NISA優先度として横に並べる設計です。
税制確認レイヤー NISA、課税口座、損出し候補、配当課税、外国税額控除、年末確認を分けて表示するデータ枠を作成しました。 反映済み 税務判断や売買指示ではなく、確認候補を表示する機能です。
高度税制確認 エンジェル税制、デリバティブヘッジ、NISA損失注意など、上級者向けの確認項目を通常候補とは別枠に分離しました。 反映済み 実行推奨ではなく、確認必須の候補として扱います。
量子コンピューター・フィジカルAI探索 既存候補とは混ぜず、量子コンピューターとフィジカルAIを新しい探索チャンネルとして作成しました。 作業中 現時点では購入候補ではありません。既存の量的スコア、決算、受注、PER/PBR/ROE等を通過した場合のみ候補化します。
リンク確認 ホーム内のリンク検査を行い、主要リンクに欠落がないことを確認しました。 確認済み GitHub Pagesへ反映済みです。

3. 量子・フィジカルAI探索枠の扱い

区分見方現在の扱い次に必要な確認
量子コンピューター 国策、研究開発、公式IRを根拠に長期テーマとして監視します。 作業中 量子関連の受注、商用サービス売上、共同研究の進捗が株価に説明可能な形で出ているか確認します。
フィジカルAI ロボット、センサー、制御機器、モーター、FA部品など、実需に近い企業群として見ます。 作業中 受注、設備投資、営業利益率、PER/PBR/ROE、最大下落率を補完して既存候補と比較します。
既存候補との関係 テーマだけで既存の購入候補10社へ混ぜることはしません。 別枠管理 量的データを通過した場合に限り、購入検討候補への昇格を検討します。

5. 未完了・今後の作業

残作業理由次の対応
量子・フィジカルAI候補の数値補完 テーマ枠は作成済みですが、各銘柄のPER/PBR/ROE、CAGR、下落率、決算・受注の確認がまだ十分ではありません。 既存100社データにある銘柄は接続し、不足銘柄は追加取得キューに入れます。
6月イベント後の再判定 6月のCPI、日銀、FOMC後に金利・為替・指数・個別株価の反応を見る必要があります。 イベント後の実数を入力し、購入検討候補を再判定します。

顧客向け画面の整理継続 機能が増えているため、毎日見る画面と補足資料画面をさらに分離する必要があります。 実用画面は「今日見る項目」「買わない条件」「次の作業」に絞ります。
