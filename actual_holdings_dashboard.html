自動更新の動き

  1. 1時間ごとに、日本株全市場の遅延価格を一括取得し、端末内で保有29銘柄だけを照合します。
  2. 取得単価と数量は正本CSVのまま固定し、現在評価額・含み損益・構成比だけを再計算します。
  3. 結果をこのHTML、最新保有表CSV、時間別価格履歴CSVへ書き出します。
  4. 取得失敗の銘柄は前回成功値を保持し、鮮度欄に「前回成功値」と表示します。

取得元はYahoo Finance JP full-market screener (delayed)です。保有銘柄コードは取得先へ個別送信しません。正式な売買判断や発注前には、証券会社画面の現在値・数量・NISA預りを再確認します。

売買管理ルール

銘柄ごとの過去60営業日の年率換算変動率を σ とし、値動きの大きさに応じて価格条件を変えます。固定の一律損切りではありません。

基準計算式用途
利益監視 P20%〜35%の範囲で 2×σ×√(60/252)取得価格からP%以上上昇すると利確準備へ移行
高値下落 T8%〜15%の範囲で 1.5×σ×√(20/252)利益監視後、高値からT%以上下落したか確認
50%売却 L18%〜30%の範囲で 3×σ×√(20/252)取得価格からL%以上下落したか確認
全売却 F25%〜35%の範囲で L+5ポイント取得価格からF%以上下落したか確認

判定の優先順位

  1. 全売却候補: 重大な事業毀損、または損失がF以上かつ20営業日でTOPIX比-12ポイント以下。F+5ポイント以上の損失は指数条件を待ちません。
  2. 50%売却候補: 損失がL以上、かつ20営業日でTOPIX比-8ポイント以下。どちらか一方だけでは売却候補にしません。
  3. 25%利確候補: 利益がP以上、高値からT以上下落、かつ20営業日でTOPIXを下回った場合。
  4. 監視: 売却・利確の価格条件だけに到達した場合、または取得後損失が8%以上の場合。50%売却条件への接近と対TOPIX差を確認します。
