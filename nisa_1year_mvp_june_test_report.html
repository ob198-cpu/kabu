1. 結論

6月テスト購入は可能です。
ただし、現時点のシステムは完成版ではなく、銘柄選定補助の試作版です。6月は全額購入ではなく、5月中に不足データ補完を進め、作成済みの市場テーマ確認表も参照しながら、6月上旬に毎日スコアを確認します。そのうえで、条件が整えば6月下旬に10〜25%以内の少額テスト購入を行う設計にします。
買う理由を数字で説明するための条件
「財務・割安・下落耐性」と「6月のCPI/FOMC/日銀を通過して市場が崩れていないか」を確認します。市場テーマ確認機能は5月中の追加実装対象ですが、現段階では検証途中のため、本レポートの購入判断根拠からは外しています。

4. 市場テーマ確認機能

1. テーマ候補AI半導体、金利上昇、円安、資源、素材、電力不足、防衛などを一覧化
2. テーマ数値SOX、SMH、米10年金利、ドル円、原油、VIX、銀行ETF、素材ETFを取得
3. テーマ強度直近変化率、金利・為替・資源・半導体指数の方向を点数化
4. 銘柄適合公開IRで確認できる事業内容・業種との対応を整理
5. テスト候補6月に確認する3〜5銘柄を出力
テスト候補 = NISA 1年保有スコア + 市場テーマ確認 + 除外リスク確認

この機能は、未来予測ではなく、市場テーマと関連指標を確認するための補助機能です。市場テーマ確認表は作成済みです。検証中のイベント反応データは、精度確認が終わるまで本レポートには掲載しません。

確認表: market_theme_check.html / 銘柄接続表: theme_stock_bridge.html

8. 最終方針

6月は本格運用ではなく、検証を兼ねた少額購入です。
5月中に「不足財務データ補完」と「市場テーマ確認機能」を実装し、6月上旬にスコアの安定性を確認します。6月中旬のCPI・日銀・FOMCを通過し、条件を満たした場合のみ、6月22日以降にNISA枠の10〜25%以内でテスト購入します。
